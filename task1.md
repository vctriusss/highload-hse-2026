# Сервис по заказу Такси

## Описание и аналоги

### Описание MVP

Онлайн-платформа для заказа такси

Предоставляет функционал как для пользователя, так и водителя

Для пользователя:
- выбор класса поездки (Эконом, Комфорт, Бизнес)
- расчет времени машрута и подачи автомобиля для каждого класса поездки
- отображение стоимости поездки с учетом surge
- показ вариантов маршрута между двумя точками
- просмотр геолокации водителя в режиме live
- чат с водителем и поддержкой

Для водителя:
- функционал смен (выход на смену, пауза, завершение)
- выдача релевантных предложений о заказах
- чат с пользователем и поддержкой
- статистика (отображение суммарного числа заказа, расчет суммарного и среднего дохода)

### Аналоги

Крупнейшие игроки на рынке заказа Такси:

## Ключевые продуктовые решения

Для наиболее корректного формирования цены используется алгоритм ценообразования используется механизм surge. Для формирования цены он использует следующие параметры:
- текущий спрос (кол-во пользователей, которые ищут заказ в один момент времени)
- количество водителей рядом с пользователем
- длину и сложность маршрута
- погодные условия
- метоположение точки маршрута (площадки для массовых мероприятий, вокзалы/аэропорты)

При этом для UX необходимо иметь возможность предоставлять детализацию расчета стоимости поездки.

Также при пиковых нагрузках необходимо иметь возможность ставить пользователей "в очередь". При появлении свободного водителя отображать предложения о заказе в порядке очереди

При этом сервис должен обеспечивать:
- минимальное время расчета стоимости поездки и времени подачи
- минимальное время поиска подходящих кандидатов-водителей для заказа
- минимальную задержку между реальной геолокацией водителя и отображаемой в приложении
- стабильность при пиковых нагрузках (когда множество людей заказывают такси в одно и то же время)

## Целевая аудитория

Целевая аудитория - все платежеспособные граждане от 18 до 60 лет, имеющие мобильное устройство с доступом в интернет.

Местоположение аудитории - пользователи сконцентрированы в мегаполисах, крупных и средних городах. Потенциально бизнес может масштабироваться и на более мелкие города. Города расположены в разных концах страны, соответственно необходимо учитывать повышенное latency для пользователей в удаленных локациях.

Основное местоположение - страны СНГ (Россия, Беларусь, Казахстан, Армения, Узбекистан). Потенциально - уход в межнар.
